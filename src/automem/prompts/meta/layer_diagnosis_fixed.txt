## Instructions

For each layer, assess whether it is functioning correctly or causing failures.
Pay special attention to **cross-layer interactions** — e.g., if encoding inserts
many units but retrieval gets nothing, the bottleneck is between store and
retrieve, not encode.

**Anti-thrash rule (Hard):** if the SAME failing task_ids recur as
`retrieval_noise` / `retrieval_miss_topk` across multiple rounds UNDER TWO OR
MORE DIFFERENT retrievers, those failures are probably NOT retrieval-fixable —
do NOT keep flip-flopping the retriever (e.g. hybrid → contrastive → hybrid).
Attribute them to Encode (unit content) or to `reasoning_error` (model), and say
so explicitly in `observation`.

Output a JSON object with this exact structure:

```json
{
  "encode": {
    "status": "ok | warning | critical",
    "observation": "1-2 sentences citing specific numbers from the metrics.",
    "suggestion": "Actionable change, or null if no issue."
  },
  "store": {
    "status": "ok | warning | critical",
    "observation": "...",
    "suggestion": "..."
  },
  "retrieve": {
    "status": "ok | warning | critical",
    "observation": "...",
    "suggestion": "..."
  },
  "manage": {
    "status": "ok | warning | critical",
    "observation": "...",
    "suggestion": "..."
  },
  "overall": "1-2 sentences: which layer is the primary bottleneck and what to fix first.",
  "priority_action": "Concrete architecture-space delta — phrased as 'retrieval: hybrid → contrastive', 'storage_routing.workflow: vector → hybrid', 'management: lightweight → json_full', or 'extract_types: [tip] → [workflow]'. extract_types MUST contain exactly one value from {tip, trajectory, workflow, shortcut, insight}. Avoid generic suggestions like 'tighten retrieval' or 'reduce top_k' (top_k is not a search-space dimension). Do NOT propose injection changes — injection is fixed at the runtime layer."
}
```

Rules:
- **Cite numbers**: reference exact values (e.g., "empty_rate=1.0", "156 units pruned", "0 retrieved out of 30 calls").
- **Cross-layer reasoning**: if encode→store works but retrieve fails, say so explicitly.
- **Status thresholds**: critical = layer is completely broken (e.g., 0% retrieval); warning = degraded; ok = working as expected.
- **Keep concise**: each observation/suggestion should be under 30 words.
- **Output strictly valid JSON** — no trailing commas, no comments.
- **Architecture-deltas only**: priority_action MUST name a concrete change in the 4-layer search space (extract_types / storage_routing / retrieval / management). Generic advice and injection changes are rejected.
- **Exactly one valid extract type**: any priority_action that introduces multiple values or a value outside {tip, trajectory, workflow, shortcut, insight} is rejected.
